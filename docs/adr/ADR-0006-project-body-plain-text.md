# ADR-0006: プロジェクト本文はプレーンテキストとし、Rich Text/HTMLを採用しない

## 状態

採用

## 背景

プロジェクト本文（`body`、最大20,000文字）の表現形式を決める必要がある。起案者が装飾された説明を
書けると表現力は上がるが、任意のHTML/リッチテキストを保存・表示すると **格納型XSS** のリスクが生じ、
サニタイズ基盤が必要になる。

選択肢:

1. **プレーンテキスト**（改行を保持して表示）
2. Markdown を保存し、レンダリングして表示
3. HTML / WYSIWYG エディタ（リッチテキスト）

## 判断

**選択肢1（プレーンテキスト）** を採用する。表示は `white-space: pre-wrap` で改行を保持する。

## 判断理由

- 格納型XSSの攻撃面を根本から排除できる。HTMLを一切解釈しないため、サニタイズライブラリ
  （DOMPurify等）や許可タグのallowlist運用が不要（選択肢3の主要リスク・コストを回避）。
- 教育用スコープでは装飾より安全性・単純さを優先する。改行保持で最低限の可読性は確保できる。
- Markdown（選択肢2）でも表示時のHTML化にサニタイズが必須で、リンク/画像の扱いなど検討事項が増える。
  リッチな表現が要件化した時点で、サニタイズ方針を含めて改めてADR化する。

## 適用範囲

- 入力: `ProjectForm` の本文は `<textarea>`（プレーンテキスト）。バックエンドは文字数上限のみ検証。
- 表示: プロジェクト詳細（SCR-011）は `<p style="white-space: pre-wrap">` で描画し、
  `dangerouslySetInnerHTML` は使用しない（コードベース全体で不使用）。

## 結果

- 本文はプレーンテキストとして保存・表示。XSSサニタイズ層は不要。
- リッチテキスト/Markdown対応は将来課題として保留（要件化時にサニタイズ方針とセットで再検討）。
